Sine Stacker User Guide

How to create nice results in no time

1.        Try scrolling through the Amplitude Delay slider range. Do the same with Sampling Frequency. Increase Spacing to get an idea of what your waves look like.

2.        Try increasing Spacing while also increasing Line Thickness.

3.        Play around with Sampling Frequency and Curve Render Mode. Toggle Render Circles and

4.        Only render circles. Try changing Spacing and Sampling Frequency to create juggling-like patterns.

5.        Try Enabling Secondary Wave. Render lines, increase Spacing, and try to shift the frequencies and amplitudes of both main and secondary wave. Decrease the Sampling Frequency.

6.        At any time, scroll through the Progress slider range to see how the curves would behave when animated (or just toggle animation).

7.        Play with Offsets. See what changing the parameters does on a dense group of simple waves. Try it while also rendering circles (you might need to decrease Sampling Frequency) to see how offsets influence the wave shape (having disabled Even Sampling).

8.        Change the speed of your animation. Watch the frame count if you're going to export the animation. Try adjusting animation speeds if the frame count is too high.

9.        Experiment with color options and opacity.

Documentation

Application overview

This application allows users to create art using stacked sine waves. You can change the wave parameters like amplitude, frequency, color and more. Some parameters can be animated to change in time. An animation can be exported as a GIF or as a sequence of PNG images.

Layout

Preview Window

Shows a preview of the output. FPS are displayed below, set to 24 FPS by default.

Wave Settings

Contains parameters of the main sine waves shown in the preview window. Spacing dictates how close the waves are rendered to each other. Amplitude Delay influences how the amplitude changes as the y-axis grows. Sampling Frequency influences sharpness of the rendered waves. Even Sampling toggles between sampling the line evenly or according to horizontal offsets (described later). The effect can become apparent when checking Render Circles and decreasing the Sampling Frequency.

Next, there is the Main Wave Settings. It can be used to set the basic wave parameters. Amplitude Speed is useful when animating the Main Wave. It dictates how many frames does it take for the animation to repeat. Progress allows the user to cycle through the animation. This way, the amplitude can be increased to some proportion of its maximum value, otherwise determined by Amplitude Multiplier. Lastly, the Animate Amplitude toggle can be used to turn the animation on and off.

Furthermore, the user can Enable Secondary Wave to create a second sine wave that is multiplied with the main wave. The settings remain the same.

Horizontal Wave Offsets

The rendered sine waves can be each offset horizontally. The offset amount for each sine wave is determined by another sine wave that can be manipulated through these settings. They are very similar to other sine wave settings with one addition: Vertical Offset can be used to shift the offset function vertically.

Render Settings

Here, the objects to be rendered are selected. Curve Fill was originally a bug, but for interesting visual results, it was made into a feature. It can, however, yield strange results and even break smooth looping. Render Circles renders circles at the sampling points (it used to be a debug feature). Render Mode switches between different ways to connect sampling points, using either linear interpolation or Catmull-Rom spline. Decrease the Sampling Frequency to test.

Shape Style

General shape style can be set here. Curve Tightness is useful when Render Mode: Curve is selected. The remaining sliders can be used to manipulate the size and opacity of rendered shapes.

Color Settings

Allows the user to change the color of the background, sine waves, wave fill and circles. Wave Hue Shift can be used to cycle through the different values of hue in the HSB color model.

Export

Offers various ways to export the artwork. When a different than the preview size is selected, unchecking the Upscale when exporting option extends the waves across the entire image.

The GIF section is used to export an animation loop. The frame count is determined by the speeds of animated components. If there is no animation, a single frame is exported.

The SVG section downloads a still image in a vector format.

The PNG section can be used to download PNGs of different sizes. PNG Sequence contains options to download the animation as a sequence of PNGs.

The Download JSON button downloads the current parameters that can be loaded by the application.

Import

Loads a JSON parameter file that was created using the Download JSON button.
